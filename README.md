# SHAZO | شازو — صيدليتك الذكية

تطبيق ويب عربي (RTL) لصيدلية حديثة، مبني بـ **Next.js 14 (App Router) + TypeScript + Tailwind CSS**.
جميع البيانات الأساسية (الأدوية، المنتجات، المقالات) بيانات تجريبية محلية (JSON)، وأي إضافة/تعديل/حذف
من **لوحة تحكم الأدمن** يُحفظ فوق هذه البيانات في `localStorage` على متصفح الزائر — الموقع بالكامل
ملفات ثابتة (Static Export) بدون أي خادم خلفي أو قاعدة بيانات مركزية.

> ⚠️ **تنويه مهم**: كل المحتوى الطبي والدوائي في هذا المشروع هو محتوى تجريبي (Demo Data) لأغراض العرض التقني فقط، ولا يمثل استشارة طبية أو صيدلانية موثوقة. لا تعتمد عليه في اتخاذ قرارات علاجية حقيقية.

## التشغيل محلياً

يتطلب Node.js 18.17 أو أحدث.

```bash
npm install
npm run dev
```

ثم افتح المتصفح على [http://localhost:3000](http://localhost:3000)

للبناء كملفات ثابتة (نفس ما يُنشر على GitHub Pages):

```bash
npm run build
```

الناتج يظهر في مجلد `out/`.

## النشر على GitHub Pages

المشروع معدّ مسبقاً للنشر التلقائي عبر **GitHub Actions** (`.github/workflows/deploy.yml`):

1. تأكد أن اسم المستودع (`repoName` داخل `next.config.mjs`) مطابق تماماً لاسم مستودعك على GitHub (حساس لحالة الأحرف).
2. ادفع محتوى هذا المشروع لفرع `main` في المستودع (اجعله في **جذر المستودع** مباشرة، وليس داخل مجلد فرعي).
3. من إعدادات المستودع: **Settings → Pages → Build and deployment → Source** اختر **GitHub Actions**.
4. أي push جديد على `main` يشغّل الـ workflow تلقائياً ويبني وينشر الموقع على
   `https://<اسم-حسابك>.github.io/SHAZO/`.

### قيود مهمة لكون الموقع Static (GitHub Pages)

- لا يوجد خادم أو قاعدة بيانات حقيقية؛ كل تعديلات الأدمن محفوظة محلياً في متصفح كل زائر فقط.
- صفحات تفاصيل الأدوية/المقالات **المُضافة عبر لوحة الأدمن** تعمل فقط عند التنقّل داخل التطبيق (عبر الروابط)،
  وليس عند فتح رابطها مباشرة أو تحديث الصفحة، لأن GitHub Pages لا يولّد صفحة HTML مسبقة لعناصر غير موجودة
  وقت البناء. الأدوية/المنتجات/المقالات الأصلية (التجريبية) تعمل بكل الحالات.
- لقاعدة بيانات حقيقية يشوفها كل الزوار (وليس فقط من أضافها)، تحتاج لاحقاً ربط خدمة خلفية مثل
  Supabase أو Firebase بدل `localStorage`.

## لوحة تحكم الأدمن

الرابط: `/admin` (أو من صفحة "المزيد" ← "لوحة التحكم (أدمن)").

- رمز الدخول الافتراضي: `shazo-admin-2026` (عدّله من `ADMIN_PASSCODE` في `lib/content.ts` قبل النشر).
- تكدر تضيف/تعدّل/تحذف أدوية، منتجات، ومقالات — التغييرات تنعكس فوراً في كل صفحات الموقع لنفس المتصفح.
- زر "استعادة الافتراضي" يمسح كل تعديلاتك ويرجّع البيانات التجريبية الأصلية.
- **تنويه أمني**: هذا القفل واجهي فقط (Client-side) لإخفاء اللوحة عن الزوار العاديين، وليس حماية حقيقية —
  لا يصلح لحماية بيانات حساسة فعلياً.

## البنية

```
app/                 صفحات التطبيق (App Router)
  page.tsx           الرئيسية (لوحة المعلومات)
  drugs/              قائمة الأدوية + [id] لصفحة التفاصيل
  prescriptions/      إدارة الوصفات (إنشاء/تعديل/حذف/بحث)
  search/             البحث المتقدم مع اقتراحات فورية
  info/               المقالات التعليمية + [id] لتفاصيل المقال
  store/              المتجر + السلة + واجهة الدفع التجريبية
  account/            حساب المستخدم
  favorites/          المفضلة (أدوية ومقالات)
  notifications/      الإشعارات
  more/                قائمة المزيد (للجوال)
  admin/              لوحة تحكم الأدمن لإدارة المحتوى

components/          مكونات واجهة قابلة لإعادة الاستخدام
context/             مزودو الحالة العامة (Theme, Toast, Content, Cart, Favorites, Prescriptions, Recent)
data/                بيانات JSON تجريبية أساسية (drugs, products, articles)
lib/                 أنواع TypeScript ودوال مساعدة (تخزين محلي، دمج محتوى الأدمن، تنسيق الأسعار والتواريخ)
```

## الميزات التقنية

- Next.js App Router مع React Server/Client Components حسب الحاجة، ومُصدَّر كملفات ثابتة (`output: "export"`).
- شاشة افتتاح تفاعلية (Intro): كبسولة تُفتح بالضغط، برق كهربائي، وتشكّل شعار SHAZO — تظهر مرة كل جلسة متصفح ويمكن تخطّيها بعد 3 ثوانٍ.
- منتقي لون ثيم حي (App Appearance) في صفحة "حسابي": يغيّر كل الأزرار والأيقونات والتمييزات فوراً عبر متغيرات CSS، ويُحفظ تلقائياً.
- Tailwind CSS بنظام ألوان مخصص (كحلي داكن، أزرق كهربائي، أبيض) مع تأثيرات زجاجية (Glassmorphism).
- خط Cairo العربي عبر `next/font/google`.
- تخزين محلي (`localStorage`) للمفضلة، السلة، الوصفات، الحساب، سجل البحث، الأدوية المشاهدة مؤخراً، وتعديلات الأدمن.
- لوحة تحكم أدمن لإدارة الأدوية/المنتجات/المقالات (CRUD) بدون خادم خلفي.
- الوضع الليلي/النهاري مع منع "الوميض" عند التحميل الأول.
- تنبيهات Toast، حالات فارغة (Empty States)، هياكل تحميل (Skeletons)، ومربعات تأكيد الحذف.
- تصميم متجاوب بالكامل: قائمة جانبية RTL على الشاشات الكبيرة، وشريط تنقل سفلي على الجوال.

## ملاحظة حول shadcn/ui

تم بناء مكونات الواجهة (البطاقات، الأزرار، الحقول، الحوارات) يدوياً باستخدام Tailwind لتقليل الاعتماديات الخارجية وضمان تشغيل المشروع مباشرة بأمر `npm install` دون الحاجة لإعداد CLI إضافي. يمكنك إضافة `shadcn/ui` لاحقاً إذا رغبت باستبدال هذه المكونات بمكوناته.
